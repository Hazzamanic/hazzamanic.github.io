Replacing orchard layout's use of the http session with the cache

The Orchard Layout's module uses the http session to store data as you edit elements. This isn't great when it times out and you lose everything you've been working on. An easy solution would be to simply extend the session timeout in the web.config but I looked at the code and it doesn't just implement the http session directly, this is orchard, it's abstracted out of course. Clearly someone thought session might not be ideal. Maybe they were thinking more along the lines of web farms and sessionless set ups but whatever.
